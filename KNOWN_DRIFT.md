# openclaw-workspace: Known Drift

## Snapshot

- Last reviewed: 2026-03-11

## Drift Register

### DRIFT-001: Runtime artifacts are not fully ignored
- Evidence: `.infra.log` and `.infra.pid` are runtime files generated by workspace bring-up.
- Impact: noisy working trees and accidental staging risk.
- Status: open

### DRIFT-002: Submodule operational drift risk
- Evidence: integration still depends on pinned `agent/` and `infra/` submodules under active development.
- Impact: local edits can diverge from pinned SHAs before an intentional sync.
- Status: open

### DRIFT-003: Legacy OpenClaw operator material remains in archive scope
- Evidence: historical docs under `agent/` still mention the retired `:18789` gateway.
- Impact: low if operators stay on the README/architecture/runbook path; medium if archive docs are treated as live guidance.
- Resolution (2026-03-11): live workspace docs, health scripts, and compose stack now point only to Oracle `:8000` plus Signal `:8080`.
- Status: mitigated on 2026-03-11

### DRIFT-004: Local-model path is optional but still present in code
- Evidence: Oracle selector/provider code retains Ollama support behind `CLOUD_ONLY=false`.
- Impact: low; this is intentional compatibility, but it remains a second code path.
- Resolution (2026-03-11): default bootstrap, compose, and tests are cloud-only-first; Ollama checks are opt-in only.
- Status: mitigated on 2026-03-11
